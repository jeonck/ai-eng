# Naïve Bayes

## I. 조건부 확률 기반의 독립성 가정, Naïve Bayes 개요

```mermaid
graph LR
    A1["특징 데이터(X)"] -- "베이즈 정리 적용" --> B1["사후 확률 계산\n(Posterior)"]
    B1 --> C1["최종 클래스 판별"]
```

**정의**: 베이즈 정리( **Bayes' Theorem** )를 적용하여 모든 특징( **Feature** )이 서로 독립이라는 '나이브(순진한)' 가정하에 데이터를 분류하는 확률 모델  

**특징**:  
( **단순성** ) 계산 복잡도가 매우 낮아 대규모 데이터셋에서도 빠른 학습과 예측 가능  
( **독립성 가정** ) 특징 간의 상관관계를 무시함으로써 고차원 데이터에서도 성능 유지 ( **Strong Independence** )  
( **소량 데이터** ) 적은 학습 데이터로도 비교적 준수한 성능을 보이며 실시간 시스템에 적합  

## II. Naïve Bayes의 수학적 원리 및 모델 종류

### 가. 베이즈 정리의 수식 표현
`P(C|X) = [P(X|C) * P(C)] / P(X)`
- **P(C|X)**: 데이터 X가 주어졌을 때 클래스 C일 확률 ( **Posterior** )
- **P(X|C)**: 클래스 C에서 데이터 X가 나타날 확률 ( **Likelihood** )
- **P(C)**: 클래스 C의 사전 확률 ( **Prior** )

### 나. 주요 확률 분포 모델

| 모델 종류 | 특징 | 주요 용도 |
| :--- | :--- | :--- |
| **Multinomial NB** | 단어의 출현 빈도를 카운트 | 텍스트 분류, 스팸 메일 필터링 |
| **Bernoulli NB** | 특징의 존재 여부(0/1)만 판단 | 짧은 문서의 이진 분류 |
| **Gaussian NB** | 특징이 정규 분포를 따른다고 가정 | 연속형 수치 데이터를 가진 분류 문제 |

## III. Naïve Bayes의 장점과 한계점

| 항목 | 상세 내용 |
| :--- | :--- |
| **핵심 장점** | 속도가 매우 빠름, 노이즈와 결측치에 강함, 다중 클래스 분류가 쉬움 |
| **핵심 한계** | 특징 간 독립성 가정이 현실과 다를 경우 성능 저하, 학습 데이터에 없는 범주는 확률이 0이 됨 |
| **해결 방안** | 확률 0 방지를 위한 **Laplace Smoothing** 적용, 도메인 지식을 활용한 변수 선정 |

**기술 동향**: 딥러닝 모델에 비해 복잡한 문맥 파악은 어렵지만, 압도적인 속도와 가성비 덕분에 여전히 스팸 필터링, 초기 텍스트 분류, 감성 분석의 베이스라인 모델로 널리 쓰임
