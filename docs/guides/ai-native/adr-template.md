# ADR 템플릿

**Architecture Decision Record** — 아키텍처 의사결정을 기록하는 문서입니다. "무엇을 했는가"가 아니라 **"왜 이 방식을 선택했는가"**를 남깁니다.

## 왜 ADR이 필요한가

AI 코딩 환경에서 ADR이 특히 중요한 이유:

- AI는 컨텍스트 없이 코드를 생성하면 과거 결정을 무시하고 다른 패턴을 사용할 수 있습니다.
- ADR을 AI에게 먼저 읽히면, 팀의 기술 결정을 일관되게 따르는 코드를 생성합니다.
- 팀원이 바뀌어도 "왜 이렇게 만들었는지"를 설명하는 문서가 됩니다.

---

## 파일 저장 위치

```
docs/decisions/ADR-0001-[결정요약].md
docs/decisions/ADR-0002-[결정요약].md
```

번호는 순차적으로 부여합니다. 한 번 부여된 번호는 변경하지 않습니다.

---

## 템플릿

아래 내용을 복사하여 `docs/decisions/ADR-000N-제목.md`로 저장하세요.

```markdown
# ADR-[번호]: [결정 사항 요약 제목]

* **상태**: [Proposed / Accepted / Superseded / Deprecated]
* **날짜**: YYYY-MM-DD
* **결정자**: [작성자 이름 또는 AI 에이전트명]

## 1. 맥락 (Context)
어떤 상황에서 이 결정이 필요하게 되었나요?
해결하려는 문제나 비즈니스 요구사항을 설명합니다.

## 2. 결정 (Decision)
우리는 [A 기술/패턴]을 사용하기로 결정했습니다.
(AI가 구현 시 반드시 지켜야 할 핵심 원칙 포함)

## 3. 대안 (Alternatives)
고려했던 다른 선택지들:
- 대안 A: [장/단점]
- 대안 B: [장/단점]

## 4. 트레이드오프 및 결과 (Consequences)
이 결정으로 얻는 이득과 감수해야 할 비용은 무엇인가요?
- **긍정적**: [예: 개발 속도 향상, 확장성 확보]
- **부정적**: [예: 학습 곡선 존재, 초기 설정 비용]

## 5. AI 가이드라인
AI가 이 결정을 바탕으로 코드를 생성할 때 주의할 점:
- "이 패턴을 적용할 때는 [X] 라이브러리 대신 [Y]를 사용하세요."
- "절대 [Z] 방식은 사용하지 마세요. 이유: ..."
```

---

## ADR 상태 정의

| 상태 | 의미 |
|---|---|
| **Proposed** | 제안됨, 아직 팀 합의 전 |
| **Accepted** | 팀 합의 완료, 현재 적용 중 |
| **Superseded** | 더 나은 결정으로 대체됨 (대체 ADR 번호 명시) |
| **Deprecated** | 더 이상 유효하지 않음 |

## 작성 예시

```markdown
# ADR-0003: 상태 관리 라이브러리로 Zustand 채택

* **상태**: Accepted
* **날짜**: 2025-05-06
* **결정자**: 개발팀 / Claude Code

## 1. 맥락 (Context)
React 앱의 전역 상태가 복잡해지면서 props drilling이 3단계 이상 발생하고 있습니다.
Redux 도입을 검토했으나 보일러플레이트가 과도하게 많습니다.

## 2. 결정 (Decision)
Zustand를 전역 상태 관리 라이브러리로 채택합니다.
모든 전역 상태는 `src/store/` 디렉토리에 도메인별로 분리합니다.

## 3. 대안 (Alternatives)
- Redux Toolkit: 생태계 풍부하나 보일러플레이트 과다
- Recoil: Atom 개념은 좋으나 Meta의 유지보수 불확실

## 4. 트레이드오프 및 결과 (Consequences)
- **긍정적**: 코드량 70% 감소, 학습 곡선 낮음
- **부정적**: Redux DevTools 수준의 디버깅 도구 미지원

## 5. AI 가이드라인
- 전역 상태는 반드시 `src/store/` 아래 도메인별 파일로 분리하세요.
- `useState`로 해결 가능한 로컬 상태는 Zustand를 사용하지 마세요.
- `create()` 함수 내부에서 직접 비동기 처리하지 말고 별도 액션 함수로 분리하세요.
```

---

## AI에게 ADR 먹이는 프롬프트

```
docs/decisions/ 디렉토리의 모든 'Accepted' 상태 ADR을 읽고,
각 결정의 AI 가이드라인을 숙지한 뒤 아래 기능을 구현해줘.
ADR에 위반되는 패턴을 사용하려 할 경우, 구현 전에 먼저 알려줘.
```
